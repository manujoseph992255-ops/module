What is DBMS?

DBMS stands for Database Management Systems.

These are systems to store, retrieve or, sometimes, manipulate data.

Developed to handle large amounts of data.

Why DBMS?

Consider a bank that maintains customer’s account details, employee details, bank device details, etc.

These details need to be stored in such a way that it can be added, deleted, updated and retrieved from one place.

DBMS is a software designed for these types of operations.

RDBMS

What is RDBMS?

RDBMS stands for Relational Database Management Systems.

RDBMS allows to store, retrieve or manipulate data, but in a more efficient way than DBMS.

Apart from rows and columns the RDBMS table has following components:

  • Domain
  • Instance
  • Schema
  • Keys

Database Keys

Keys

A key is a data item (one or more columns) that allows you to uniquely identify a record.

Keys are used to fetch a specific record and provide useful constraints such as "Unique value" which ensures no two rows have the same value.

Types of Keys

Database supports many types of keys:

  • Candidate key
  • Primary key
  • Foreign key
  • Unique key
  • Alternate key

Candidate Key vs Primary Key

Candidate Key: An attribute (column) or a set of attributes that uniquely identifies a record.

Example: Customer ID + Store ID + Location ID in a customer transaction table.

Primary Key: Identifies each record in a table and must never be the same for two records in that table.

Example: Customer ID identifying a customer uniquely in a customer table.

Key Differences:

Candidate KeyPrimary Key
Any column or combination that can qualify as a unique key.A column or combination that uniquely identifies a record.
There can be multiple Candidate Keys in one table.There is only one Primary Key in a table.
Each Candidate Key can qualify as a Primary Key.(The chosen identifier)

Primary Key Rules

  • Must uniquely identify each row.
  • Cannot contain NULL values.
  • Only one primary key per table.
  • Can be a single column or multiple columns (known as a composite key).

Foreign Key

Foreign keys are the columns of a table that point to the primary key of another table.

Example: CustomerID in a customer transaction table points to the CustomerID in the main customer table.

Alternate and Unique Keys

Alternate Key: A candidate key that is not chosen as the primary key.

Example: Store Name + Store Location in a store information table.

Unique Key: An attribute or set of attributes to uniquely identify a record.

This is similar to a primary key but can contain a null value.

Example: Store Name in a store information table.

Unique Key vs Primary Key

Unique KeyPrimary Key
Must be unique.Must be unique.
Can contain NULL values.Cannot be NULL.
Multiple unique keys allowed per table.Only one primary key allowed per table.

Database Normalization

What is Normalization?

Normalization is the process of organizing data in a database.

Goals:

  • Reduce data redundancy (duplicate data).
  • Improve data consistency.
  • Prevent update, insert, and delete errors (anomalies).
  • Ensures the database is efficient, logical, and easy to maintain.

What Problems Does Normalization Solve?

Normalization avoids common data issues known as anomalies:

  • Update Anomaly: If you store DeptName in an Employee table and change it in one row, other rows might still have the old value.
  • Insert Anomaly: You cannot add a new department because no employee exists for it yet.
  • Delete Anomaly: If the only employee in a department leaves, the information about the department itself is lost.

What is a Normal Form?

A Normal Form is a set of rules a table must follow to reduce redundancy.

Each level increases the quality of the table design.

Most real-world databases follow 1NF, 2NF, and 3NF.

Definition: A table is in a particular Normal Form if it satisfies all the rules of that form.

First Normal Form (1NF)

A table is in 1NF if:

  • All values are atomic (no multiple values in one cell).
  • There are no repeating groups or columns.
  • Each row is unique.

Example of NOT 1NF (Non-Atomic):

StudentIDNamePhoneNumbers
101John98765, 98450

Example of 1NF (Atomic):

StudentIDNamePhoneNumber
101John98765
101John98450

Second Normal Form (2NF)

A table is in 2NF if:

  • It is already in 1NF.
  • No Partial Dependency: No non-key column should depend on only part of a composite primary key.

Convert to 1NF (Make values atomic)

StudentIDNamePhoneNumber
101John98765
101John98450
102Maya91234

Example of a Table NOT in 2NF

Orders Table:

OrderIDProductIDProductNameQuantity
1P1Keyboard2
1P2Mouse1
2P1Keyboard3

ProductName does NOT depend on OrderID.

It depends ONLY on ProductID → partial dependency.

✗ Therefore, table is NOT in 2NF.

Convert to 2NF

We split the table into 2 tables:

Orders Table (only order-related data):

OrderIDProductIDQuantity
1P12
1P21
2P13

Product Table (product-related data):

ProductIDProductName
P1Keyboard
P2Mouse

After Splitting

  • ✓ No repetition of product names
  • ✓ ProductName now depends ONLY on ProductID
  • ✓ Orders table only stores order data
  • ✓ No partial dependency

BOTH tables are now in 2NF.

Third Normal Form (3NF)

A table is in 3NF if:

  • It is already in 2NF.
  • No Transitive Dependency: Non-key columns must not depend on another non-key column.

Example of NOT 3NF (Employee Table)

EmpIDEmpNameDeptIDDeptName
1RiyaD1HR
2JohnD2Finance
3MeeraD1HR

Problem: DeptName depends on DeptID, a non-key column.

This is transitive dependency → breaks 3NF.

Convert to 3NF

Employee Table:

EmpIDEmpNameDeptID
1RiyaD1
2JohnD2
3MeeraD1

Department Table:

DeptIDDeptName
D1HR
D2Finance

No transitive dependency. Table is in 3NF.

Structured Query Language (SQL) Basics

The commands available in SQL can be broadly categorised as follows:

  • Data Definition Language (DDL)
  • Data Manipulation Language (DML)
  • Data Query Language (DQL)
  • Data Control Language (DCL)
  • Transactional Control Language (TCL)

Types of SQL Commands

  • TCL: Set Transaction, Save point, Roll Back, Commit
  • DQL: Select
  • DML: Delete, Update, Insert
  • DDL: Drop, Alter, Create, Truncate
  • DCL: Revoke, Grant

Data Definition Language (DDL)

A database is a collection of many tables, and a database server can hold many of these databases.

Hierarchy

Database Server → Databases → Tables (defined by columns) → Rows

Databases and tables are referred to as database objects.

Any operation, such as creating, modifying, or deleting database objects, is called Data Definition Language (DDL).

DDL is used to create a new schema as well as to modify an existing schema.

The typical commands available in DDL are: CREATE, ALTER, DROP, TRUNCATE.

Create Database (DDL - CREATE DATABASE)

The CREATE DATABASE statement is used to create a new SQL database.

Syntax:

CREATE DATABASE databasename;

The semicolon character (;) is a SQL statement terminator.

To create tables in the database you need to first select the database.

Syntax to select the database:

USE databasename;

DDL - CREATE DATABASE - Example

We will create a database called company:

CREATE DATABASE company;

Use SHOW DATABASES; to check if the database "company" has been CREATED.

The database company is now created.

The DROP DATABASE statement is used to drop an existing SQL database.

Creating Tables & Data Types

Prerequisites for Creating Tables

To create and maintain a table, you need a database.

While defining columns in a table, you should mention:

  • The name of the columns
  • The datatype (integer, floating point, string, and so on)
  • Default value (if any)

Let’s take a look at data types before we create a table.

Data Types - Built-in

Every relational database has its own maximum and minimum size limits for different data types; you don’t need to remember the limits. The main categories are:

  • Numeric: INT, BIGINT, BIT, etc.
  • Floating: DECIMAL, FLOAT, DOUBLE
  • Strings: CHAR, VARCHAR, TEXT
  • Date/Time: DATE, TIMESTAMP, YEAR

Creating Tables (CREATE TABLE)

The CREATE TABLE statement is used to create a new table in a database.

Syntax:

CREATE TABLE table_name (
    column1 datatype,
    column2 datatype,
    column3 datatype,
    ....
);
  • The column parameters specify the names of the table's columns.
  • The datatype parameter specifies the type of data the column can hold (e.g., varchar, integer, date, etc.).

CREATE TABLE - Example

We will create a customers table to hold information like First Name, Last Name, and Country.

Implementation Steps:

  1. Select the database: USE company;
  2. Execute the CREATE TABLE command:
CREATE TABLE customers (
    CustomerId int,
    first_name varchar(20),
    last_name varchar(20),
    country varchar(20)
);

Check the table definition using: describe customers;

Modifying Tables (ALTER TABLE)

The ALTER TABLE statement is used to add, delete, or modify columns in an existing table. It is also used to add and drop various constraints.

ADD Column

To add a column in a table, use the following syntax:

ALTER TABLE table_name ADD column_name datatype;

Example: Add an email column to the customers table.

ALTER TABLE customers ADD email varchar(255);

DROP Column

To delete a column in a table, use the following syntax:

ALTER TABLE table_name DROP COLUMN column_name;

Example: Remove the email column from the customers table.

ALTER TABLE customers DROP COLUMN email;

Data Manipulation Language (DML)

DML is used to retrieve, store, modify, delete, insert, and update data in a database.

The typical commands available in DML are:

  • INSERT: Add new records.
  • UPDATE: Modify existing records.
  • DELETE: Remove records.

INSERT INTO Syntax

The INSERT INTO statement is used to insert new records in a table.

Option 1: Specifying Columns (Recommended)

INSERT INTO table_name (column1, column2, ...) VALUES (value1, value2, ...);

Option 2: All Columns (Order must match)

INSERT INTO table_name VALUES (value1, value2, value3, ...);

INSERT - Example

Insert a record into the customers table:

INSERT INTO customers (CustomerId, first_name, last_name, country)
VALUES (1, 'John', 'Doe', 'USA');

DROP vs TRUNCATE TABLE

DROP TABLE

The DROP TABLE statement is used to permanently remove an existing table and all its data from a database.

Syntax: DROP TABLE table_name;

WARNING: Be careful! Deleting a table results in the loss of all information stored in it!

TRUNCATE TABLE

The TRUNCATE TABLE statement is used to delete the data inside a table, but the table structure (columns) remains.

Syntax: TRUNCATE TABLE table_name;
