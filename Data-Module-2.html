1. Data Manipulation

Concept

Data manipulation refers to the process of modifying, organizing, and preparing data so it can be analyzed effectively. It is an essential step before performing data analysis because raw data often contains:

  • missing values
  • duplicate records
  • inconsistent formats
  • errors
Data manipulation ensures the dataset becomes accurate, consistent, and usable.

Example of Raw Data (With Errors):
ID Name Age City Status
101 Arjun 28 Kochi Active
101 Arjun 28 Kochi Active (Duplicate)
102 Sarah NULL Dubai Inactive
103 John 35 new york Active
104 M@ike twenty London Pending

Notice: 101 is a duplicate, Sarah has a missing value, "new york" is inconsistently formatted, and Mike has special character errors.

2. ETL Process

ETL stands for Extract, Transform, and Load. It is a common process used to move and prepare data for analysis.

  • 2.1 Extract: Data is collected from various sources (databases, files, websites) and stored in a staging area.
  • 2.2 Transform: Data is cleaned and converted (removing duplicates, correcting errors, converting formats).
  • 2.3 Load: Cleaned data is stored in a target system like a data warehouse or database.
Insight:

ETL ensures that data from different sources becomes consistent and ready for analysis.

3. Common Data File Formats

Data is often stored and transferred using common file formats:

  • CSV (Comma Separated Values): Data separated by commas, used for spreadsheets.
  • XML (Extensible Markup Language): Uses tags to describe data structure.
  • JSON (JavaScript Object Notation): Stores data in key-value pairs.
Format Examples: 
// CSV
Name,Age,City
Arjun,28,Kochi

// XML
<name>Arjun</name>

// JSON
{ "name": "Arjun", "age": 28 }

4. Data Cleaning

Data cleaning is the process of identifying and correcting errors or inconsistencies to ensure data is accurate, complete, and consistent.

Before vs. After Cleaning:
Issue Type Raw Data (Dirty) Cleaned Data (Result)
Duplicate ID: 101, Name: Arjun (Duplicate) ID: 101, Name: Arjun (Unique)
Missing Age: NULL Age: 28 (Mean)
Formatting City: new york City: New York
Special Chars Name: M@ike Name: Mike

Cleaning transforms uncleaned, raw datasets into reliable information for analysis.

4.1 Handling Missing Values: Occur when data is not recorded (e.g., NULL). Can be handled by replacing with mean/median or removing rows.

5. Organizing Data

Methods to make data easier to analyze include:

  • 5.1 Sorting: Arranges data in ascending or descending order (e.g., sorting by salary).
  • 5.2 Filtering: Selects records based on conditions (e.g., salary > 50,000).
  • 5.3 Transposing: Changes rows into columns or vice versa.

6. Aggregating Data

Aggregation combines multiple values to produce summary results. Key operations:

  • 6.1 Sum: Calculates the total value (e.g., total sales per region).
  • 6.2 Count: Counts the number of records (e.g., number of employees per department).
  • 6.3 Average: Calculates the mean value (e.g., average salary per department).
