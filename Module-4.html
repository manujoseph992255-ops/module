Introduction to Code Structure

As programs become larger, writing code in a structured and organized way becomes very important.

Good programs should be:

  • Easy to read
  • Easy to maintain
  • Properly documented
  • Reusable

Professional developers write code so that other developers can understand it easily.

Two important practices help achieve this:

Code Documentation

2️⃣ Functions

4.1 Code Documentation

Documentation explains what the code is doing.

It helps:

  • Team members understand the program
  • Maintain large systems
  • Debug errors faster
  • Improve readability

Python supports documentation using:

  • Indentation
  • White space
  • Comments
  • Docstrings

Indentation

Python uses indentation to define code blocks.

Incorrect indentation leads to errors.

Example

marks = 70
if marks >= 50:
    print("Pass")

Output

Pass
Explanation
The indented line belongs to the if block.
Python does not use braces { } like other languages.
Instead, indentation defines the structure.

Introduction to Documentation

Documentation is as important as the code itself. It explains what the code does, why it exists, and how to use it.

Good documentation helps:

  • Other developers understand your logic
  • You remember what you wrote months later
  • Automated systems generate instruction manuals
Business Importance:
• Large organizations rely on clear documentation to hand over projects
• API documents allow different companies to connect their software
• Compliance reports are generated from documented code

Incorrect Indentation

Example

if marks >= 50:
print("Pass")

Error

IndentationError
Explanation
Python requires consistent indentation to determine program structure.

White Space

White space improves code readability.

Bad style

x=10+5*2

Better style

x = 10 + 5 * 2
print(x)

Output

20
Explanation
Spaces around operators make the code cleaner and easier to read.

Comments

Comments are used to explain code logic.

Python ignores comments during execution.

Example

# Calculate total sales
sales = [5000, 7000, 6000]
total = sum(sales)
print(total)

Output

18000
It helps other programmers understand the purpose of the code.

Coding Practice: Comments

Context: Internal script labeling for a marketing agency audit.

Docstrings

Docstrings describe what a function does.

They are written using triple quotes.

Example

def square(n):
    """Return the square of a number"""
    return n * n

print(square(5))

Output

25
Explanation
The triple quotes """ """ store documentation for the function.

Accessing Docstrings

Docstrings can be accessed programmatically.

Example

print(square.__doc__)

Output

Return the square of a number
This allows Python tools to generate documentation automatically.

Coding Practice: Docstrings

Objective: Technical documentation for a social media engagement counter.
Challenge: Define a function total_likes() that includes a Docstring and return 0.

Using pydoc for Documentation

Python provides a tool called pydoc.

Example

import pydoc
print(pydoc.render_doc(square))
Explanation
pydoc automatically generates documentation for functions, modules, and classes.
This is useful in large professional projects.

4.2 Functions

Functions organize code into reusable blocks.

Benefits of functions:

  • Avoid repeating code
  • Improve readability
  • Simplify debugging
  • Make programs modular

Defining a Function

Functions are defined using the def keyword.

Example

def greet():
    print("Welcome to the finance system")

greet()

Output

Welcome to the finance system
Explanation
def defines a function.
greet() calls the function.

Coding Practice: Define Function

Task: Automated status-verification routine for an enterprise server cluster.
Challenge: Define a function show() that prints "Visible".

Function with Parameters

Functions can accept input values called parameters.

Example

def calculate_sales(price, quantity):
    total = price * quantity
    print("Total Sales:", total)

calculate_sales(50, 10)

Output

Total Sales: 500
Explanation
price and quantity are parameters.
They allow the function to work with different inputs.

return Statement

Functions can return values using return.

Example

def profit(revenue, cost):
    return revenue - cost

result = profit(10000, 7000)
print("Profit:", result)

Output

Profit: 3000
The function calculates profit and returns the result to the calling program.

Default Parameters

Functions can have default values.

Example

def greet(name="Customer"):
    print("Welcome", name)

greet()
greet("Rahul")

Output

Welcome Customer
Welcome Rahul
Explanation
If no argument is provided, Python uses the default value.

pass Statement in Functions

Sometimes we define a function but do not implement it yet.

Example

def future_feature():
    pass

print("Function created successfully")

Output

Function created successfully
Explanation
pass prevents syntax errors when a function body is empty.

Integrated Business Example

Example: Calculating average sales.

def calculate_average_sales(sales_list):
    """Return the average sales value"""
    
    total = sum(sales_list)
    average = total / len(sales_list)
    
    return average

sales = [5000, 7000, 9000]
avg = calculate_average_sales(sales)
print("Average Sales:", avg)

Output

Average Sales: 7000.0
Explanation
This program demonstrates:
• Function creation
• Parameters
• Return values
• Documentation using docstrings
